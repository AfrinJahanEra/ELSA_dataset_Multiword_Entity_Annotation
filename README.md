# ELSA_dataset_Multiword_Entity_Annotation

A complete NLP pipeline for **Bangla (Bengali) product review analysis** — covering multi-word entity annotation, Named Entity Recognition (NER), Sentiment Analysis, and Zero-Shot Classification using state-of-the-art transformer models on the **ELSA 10K dataset**.

---

## Project Overview

This repository implements a full annotation-to-training pipeline for extracting **multi-word product entities** from Bangla e-commerce reviews:

1. **Dataset Annotation** — Custom BIO tagging scheme to detect and annotate multi-word product entities from the raw ELSA dataset.
2. **Synthetic Data Generation** — Augmented training data produced by multiple LLMs (Gemini, GPT, Claude, DeepSeek).
3. **Model Training** — Fine-tuning three multilingual transformer models for NER and Sentiment Analysis.
4. **Zero-Shot Baseline** — Multilingual NLI-based zero-shot classification for comparison.
5. **Evaluation & Reporting** — Comprehensive metrics, confusion matrices, and structured report export.

### Models

| Model | HuggingFace ID | Architecture |
|---|---|---|
| **BanglaBERT** | `csebuetnlp/banglabert` | ELECTRA |
| **mBERT** | `bert-base-multilingual-cased` | BERT |
| **XLM-RoBERTa** | `xlm-roberta-base` | RoBERTa |

### Tasks

- **NER** — 3-label BIO scheme: `O`, `B-Product`, `I-Product`
- **Sentiment Analysis** — 3-class: Negative (0), Neutral (1), Positive (2)
- **Zero-Shot Classification** — NLI-based sentiment prediction without task-specific fine-tuning

## Getting Started

### 1. Prerequisites

- Python 3.10+
- PyTorch (with CUDA for GPU training)

### 2. Install Dependencies

```bash
pip install transformers datasets evaluate accelerate>=1.1.0
pip install torch sentencepiece seqeval
pip install pandas numpy matplotlib seaborn scikit-learn
```

### 3. Dataset Annotation (BIO Tagging)

Run the multi-word entity annotation script on the raw ELSA dataset:

```bash
python Dataset_analysis/elsa_multiword_entity_bio_v3.py
```

This produces:
- BIO-tagged sentences with `B-Product` / `I-Product` / `O` labels
- Multi-word entity reports with frequency statistics and sentiment breakdowns

### 4. Generate Notebooks

All three model notebooks are generated from a single template:

```bash
python _build_nb.py
```

This creates three `.ipynb` files in `Codes/`, each containing:

| Section | Description |
|---|---|
| A–B | Install & Import libraries |
| C | Load synthetic training + test data |
| D | Parse BIO tags into tokens & labels |
| E–F | Define label mappings & 80/20 train-val split |
| G | Tokenizer setup with sub-word label alignment |
| H | Fine-tune NER (token classification, 3 epochs) |
| I–J | Fine-tune Sentiment Analysis (sequence classification, 3 epochs) |
| K | Evaluate SA on test set |
| L | End-to-end inference example |
| M | Zero-shot classification (mDeBERTa-v3 multilingual NLI) |
| N | Save full report to `output_txt/` |

### 5. Run a Notebook

Open any notebook in Jupyter or VS Code and run all cells:

```bash
jupyter notebook Codes/BanglaBERT_GEMINI_FLASH_EXTENDED.ipynb
```

> **Note:** Notebooks use relative paths (`../Synthetic Datasets/`) since they run from the `Codes/` directory.

## Notebook Pipeline Details

### NER — Token Classification

- Merges the 80/20 train+val splits for final training (3000 samples)
- Uses `seqeval` for token-level precision, recall, F1, and accuracy
- Saves the best checkpoint (by validation F1) to `ner_{model}_final/`

### Sentiment Analysis — Sequence Classification

- 3-class: Negative / Neutral / Positive
- Best checkpoint selected by validation accuracy
- Evaluated on the 30-sample held-out test set

### Zero-Shot Classification

- Model: `MoritzLaurer/mDeBERTa-v3-base-xnli-multilingual`
- Candidate labels: `["negative", "neutral", "positive"]`
- Provides a no-training baseline for comparison against fine-tuned models

## Synthetic Datasets

All synthetic datasets contain 3000 Bangla review samples with columns:

| Column | Description |
|---|---|
| `id` | Row identifier |
| `review` | Raw Bangla review text |
| `sentiment` | Label: 0 (Negative), 1 (Neutral), 2 (Positive) |
| `bio_tagged_review` | Review with BIO entity tags (`token-B`, `token-I`) |

The **Gemini 3.5-Flash** dataset is used as the primary training set across all three model notebooks.
